两个洞:veth 凿 net 墙 · volume 凿 mnt 墙

墙是默认,洞是例外。每个洞都凿在某一扇 namespace 墙上,是"受控连通"
洞一 · veth(凿 net 墙)
容器 netns_A
eth0 = veth 一头
▲ priv->peer ▼  (一对一互指,RCU)
宿主 netns
veth 另一头 → 插 docker0
// 配对:veth_newlink() rcu_assign_pointer(priv->peer, peer); rcu_assign_pointer(peer_priv->peer, dev); // 传输:veth_xmit() rcv = rcu_dereference(priv->peer); veth_forward_skb(rcv, skb, …); // 直接喂对端
无物理介质,把 skb 从一头递到另一头(零拷贝)
洞二 · volume(凿 mnt 墙)
容器 mnt 窗(挂载树)
/  ←─ overlay merged(易失,走 copy_up) ├─ /usr ... ├─ /etc ... └─ /data  ←─ bind mount 竖井 │  直通宿主,绕过 overlay ▼ 宿主 /var/lib/docker/volumes/db/_data(持久)
// mount --bind /host /data do_mount → path_mount └─ do_loopback() └─ clone_mnt() // 新 mount,指向同一 dentry └─ graft 进容器挂载树的 /data 点
挂载树里加一个直通宿主的入口,读写不进 overlay
veth = 在 net 墙上接一根"穿墙网线"(peer 指针);volume = 在 mnt 墙上凿一口"直通宿主的竖井"(bind mount)。隔离默认,连通显式。
绿=net 洞(veth) · 蓝=mnt 洞(volume) · 源码:drivers/net/veth.c(veth_newlink/veth_xmit)· fs/namespace.c(do_loopback/clone_mnt)
