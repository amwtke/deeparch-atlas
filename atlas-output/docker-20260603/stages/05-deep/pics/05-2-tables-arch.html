cgroup 怎么实现 cpu / memory / io 三张表

接口即文件:写一个目录里的文件 = 设表盘;上户口 = 写 cgroup.procs;记账即配额:每次消费都 charge
/sys/fs/cgroup/demo/  (一个目录 = 一个户头)
echo "50000 100000" > cpu.max    echo 100M > memory.max    echo "8:0 wbps=10485760" > io.max
echo $$ > cgroup.procs  ←  把进程(及其后代)上户口,从此每笔消费都记这个账
▼   内核为每个 controller 在这个 cgroup 上挂一个 css(cgroup_subsys_state)   ▼
cpu 表(电表)
cpu.max = "50000 100000"
每 100ms 给 50ms = 半颗
CFS bandwidth · kernel/sched/fair.c
每次 tick:__account_cfs_rq_runtime() runtime_remaining -= 已运行; if (runtime_remaining <= 0) throttle_cfs_rq(); // 踢出队列
超额 → 限速(throttle)
本周期不再跑,下周期续
memory 表(煤气表)
memory.max = 100M
硬上限
memcg · mm/memcontrol.c
每分配页:try_charge_memcg() page_counter_try_charge(memory,…) 失败→try_to_free_mem_cgroup_pages 仍失败→mem_cgroup_oom()
超额 → OOM kill(处决)
SIGKILL · Exited 137
io 表(水表)
io.max = "8:0 wbps=10485760"
设备 8:0 写 10MB/s
blk-throttle · block/blk-throttle.c
每个 bio:__blk_throtl_bio() tg_within_bps/iops_limit()? 否→tg_dispatch_time() 算等待 →bio 进队列,定时器到点再发
超额 → 限速(排队)
IO 被推迟,不丢不杀
三张表都靠"记账":CPU 记时间片、内存记页、IO 记字节。区别在超额脾气 —— CPU/IO 降速,内存处决(超了无法"慢慢用")。
蓝=cpu(CFS) · 红=memory(memcg,会杀) · 绿=io(blk-throttle) · 源码:kernel/sched/fair.c · mm/memcontrol.c · block/blk-throttle.c(torvalds 主线)
