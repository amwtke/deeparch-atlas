clone() 一次系统调用,怎么在内核里造出"六面墙"

主线:sys_clone → copy_process() →【先建 user 窗(权限上下文)】+【copy_namespaces 派生其余五扇】→ 挂上 task
用户态:clone(CLONE_NEWPID|NEWNET|NEWNS|NEWUTS|NEWIPC|NEWUSER, …) / unshare(…)
runc 真正调它的地方 = nsexec.c 的 clone_parent()
▼
kernel/fork.c  copy_process()
复制一个新 task,其中两步专管"墙",顺序关键:copy_creds(左)+ copy_namespaces(右)
▼
① user 窗 · 特殊,必须最先建
copy_creds() → 若 CLONE_NEWUSER → create_user_ns()
ns->level = parent->level + 1; // 可嵌套 ns->owner = current uid; // uid_map/gid_map 初始为空待写 // → 容器 root ↔ 宿主普通用户
为什么先建?它是其余窗 ns_capable(CAP_SYS_ADMIN) 权限检查的"上下文"
copy_namespaces() · kernel/nsproxy.c
if (!(flags & (CLONE_NS_ALL & ~CLONE_NEWUSER))) get_nsproxy(old_ns); return 0; // 快路径:不要新窗→复用 else if (!ns_capable(user_ns, CAP_SYS_ADMIN)) return -EPERM; // 没权限不准开窗
CLONE_NEWUSER 被 & ~ 抠掉 → user 窗不归这里管(铁证)
▼
create_new_namespaces():顺序派生,每个产出一扇窗
▼
copy_mnt_ns
mnt 窗
文件系统视图
copy_utsname
uts 窗
主机名/域名
copy_ipcs
ipc 窗
信号量/共享内存
copy_pid_ns ★
pid 窗
PID 1 幻觉源头
copy_cgroup_ns
cgroup 窗
(内核额外)
copy_net_ns
net 窗
洞二 veth 凿这扇
回滚链:out_time → out_net → out_cgroup → out_pid → out_ipc → out_uts → out_ns ——任一步失败,前面全部 put 掉(all-or-nothing)
pid 窗细看 · create_pid_namespace()(PID 1 幻觉的真身)
unsigned int level = parent_pid_ns->level + 1; // 嵌套层级 if (level > MAX_PID_NS_LEVEL) goto out; // 深度上限 idr_init(&ns->idr); // 每窗自带发号器,从 1 发 ns->parent = get_pid_ns(parent_pid_ns); // 指回父,形成树
同一进程在 struct pid 的 numbers[] 里"每层一个号":容器里 1,宿主上 12345 —— 一个 idr_init,两个世界
▼
产物:struct nsproxy(装六个指针的容器)
nsproxy { mnt_ns, uts_ns, ipc_ns, pid_ns_for_children, cgroup_ns, net_ns } // user_ns 不在这里 —— 它挂在 task 的 cred 上(所以"特殊")
▼
tsk->nsproxy = new_ns;  → 这个 task(及其后代)从此"住进"这组窗
墙的全部物理实体 = task_struct 上这几个指针 + cred 里的 user_ns。没有别的。
一句话:clone "造墙" = 让新 task 的指针指向一组"独立视图对象"。改几个指针 = 一面墙。

这就是"骗进程比骗 OS 便宜三个数量级"的源码级证据 —— VM 要虚拟一整台机器,容器只要给 task 换几个指针。

橙 = namespace 派生 · 紫 = user 窗(特殊) · 蓝 = nsproxy 产物 · 绿 = 挂到 task · 金 = pid 窗(PID 1)· 红 = 失败回滚链
